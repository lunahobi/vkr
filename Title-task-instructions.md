## Таблица «Используемые технологии и платформы»

**Серверная часть:** FastAPI (Python 3.x), SQLAlchemy 2.x, Alembic, Pydantic v2, Uvicorn; REST API `/api/v1`; JWT (access/refresh); фильтрация по идентификатору организации-арендатора.

**СУБД:** PostgreSQL 16 (эксплуатационный контур Docker Compose); SQLite (локальная разработка и быстрые прогоны).

**Клиентская часть (web):** React 18, TypeScript 5.x, Vite; единый слой запросов к API; заголовки `Authorization: Bearer`, `X-Tenant-Code`.

**Клиентская часть (mobile):** Flutter / Dart; экраны слушателя (курсы, адаптивный тест, рекомендации); параметр сборки `API_BASE`.

**Инфраструктура:** Docker Engine + Docker Compose; контейнеры `db`, серверной части, web; тома для PostgreSQL и медиа.

**Канал:** HTTP/HTTPS, JSON, CORS (настраивается для эксплуатации).

## Решаемые задачи (кратко)

1. Анализ предметной области корпоративного обучения и сравнение LMS-аналогов по формальным критериям и весам.  
2. Формирование требований, сценариев использования и критического пути пользователей.  
3. Проектирование архитектуры модульного монолита, инфологической и даталогической моделей БД.  
4. Реализация серверной части (аутентификация, роли, курсы, уроки, тесты, адаптивная логика, аналитика).  
5. Реализация веб-панели администратора/преподавателя и мобильного клиента слушателя.  
6. Описание развёртывания (`docker compose`), опытной эксплуатации, информационной безопасности и экономического обоснования.  
7. Подготовка приложений: ТЗ (ГОСТ 19.201), ПиМИ (ГОСТ 19.301), опросные листы, руководства, листинги кода.
